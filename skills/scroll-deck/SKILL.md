# Scroll Deck Skill

**목적: PPT(PPTX) 를 입력으로 받아, 발표용 스크롤 HTML 로 만든다. 그게 전부다.**

원본 PPT 의 UI 를 그대로 복제하지 않는다. 슬라이드의 의도를 읽고 **고객 전달용 HTML 로
다시 구성**한다. 좌측에 슬라이드 목차, 위 → 아래로 스크롤되며 설명하는 한 장짜리 웹페이지.

> 파이프라인: **① PPT 분석 → ② 스크린샷으로 레이아웃·내용 파악 → ③ 모델(나)이 각 슬라이드를
> 깔끔한 HTML 로 재구성 → ④ 생성기가 목차·스크롤·영상 셸에 조립.**

생성기: `scripts/build_scroll_deck.py` (순수 Python + `python-pptx`, 새 의존성 없음)

## 작업 방식

1. **분석**: PPTX 본문과 슬라이드 스크린샷(`slide-NN.png`)으로 각 장표의 레이아웃·강조점·
   상세 콘텐츠와 정확한 텍스트를 파악한다.
2. **재구성**: 각 슬라이드를 `slides/NN.html`(2자리 zero-pad) 에 자체 완결형
   `<section class="frag ...">` HTML 로 작성한다. 원본을 억지로 모사하지 말고, 발표·고객
   전달에 맞게 깔끔히 재배치한다. 제목은 프래그먼트 안에서 직접 표현(자동 카드 헤더 없음).
   제품·기능명은 영문 유지.
3. **빌드**: 생성기를 실행하면 프래그먼트가 슬라이드 본문으로 삽입되고
   (`<div class="slide-wrap" id="slide-N">{프래그먼트}</div>`), 목차·스크롤·영상 셸이 조립된다.

> 스크린샷이 없으면 PPTX 본문만으로 작성한다. 프래그먼트가 없는 슬라이드는 생성기가
> PPTX 에서 본문을 자동 추출해 폴백 렌더한다(차선책 — 가능하면 프래그먼트로 재구성).

**원본보다 빈약하면 안 된다.** 텍스트만 복붙하지 말고, 원본의 불릿·수치·흐름을 모두
살리거나 확장한다. 제품 UI 스크린샷·다이어그램처럼 **글로는 전달이 약한 장표는 PPT 의
실제 이미지를 가져와 임베드**하고, HTML 인 만큼 **스크롤 등장 애니메이션·hover 등 가벼운
인터랙션**을 더해 자연스럽게 만든다.

## PPT 이미지 임베드

제품 화면·대시보드·다이어그램 등 **꼭 필요한 이미지만** 골라 가져온다(꾸밈용 배경·작은
아이콘은 제외 — 아이콘은 이모지/칩으로 대체).

```bash
python3 scripts/extract_slide_media.py <pptx> <릴리즈>/slides/img
```

- 슬라이드별 그림을 `img/sNN-k.webp`(최적화 WebP)로 추출. 배경(면적 과대)·초소형 그림은 건너뛴다.
- 프래그먼트에서 상대경로로 참조: `<figure class="fshot fshot--zoom reveal">
  <img src="img/sNN-1.webp" loading="lazy"><figcaption>…</figcaption></figure>`.
- 빌드 시 생성기가 `slides/img/` 를 출력 `<slug>/img/` 로 복사하므로 상대경로가 그대로 해석된다.
- 불필요하게 추출된 그림은 지운다(쓰는 것만 남긴다).
- **이미지가 작아 보이지 않게**: 이미지+텍스트 2단은 `.fgrid.c2` 대신 **`.fgrid.imgR`**(이미지 오른쪽)
  /**`.fgrid.imgL`**(이미지 왼쪽)을 써 이미지 칸을 더 넓게(약 60%) 잡는다. `.fshot--zoom` 이미지는
  **클릭하면 라이트박스로 전체 확대**(자동, ESC/클릭 닫기)되므로 작은 화면에서도 디테일을 볼 수 있다.

## 복잡한 다이어그램 (Mermaid · 방사형)

플로우·아키텍처·피드백 루프 등 패널/`.fflow` 로 깔끔하게 표현하기 어려운 복잡한 다이어그램은
**널리 쓰이는 라이브러리로 재구성**한다. 원본을 그대로 베끼지 말고 핵심 구조(흐름·반복·관계)를
읽어 깔끔하게 다시 그린다.

- **Mermaid**(생성기가 CDN 로드·브랜드 테마 적용): `<div class="fdiagram"><pre class="mermaid">…</pre></div>`
  안에 flowchart/sequence 등을 쓴다. `.mermaid` 가 페이지에 있을 때만 로드되고, 다크 테마·반응형
  SVG·`max-width:100%` 가 적용된다.
  - 주의: `style` 지시문 색상에 **`rgba(...)` 금지**(쉼표가 파서를 깨뜨림) → 8자리 hex(`#8b5cf622`) 사용.
  - `end` 는 예약어 → 노드 id 로 쓰지 않는다.
  - 노드가 많으면 가로로 과하게 펼쳐져 **화면에서 너무 작게** 보인다 → **고수준 루프/흐름만
    다이어그램**으로 그리고(결정 분기·반복은 점선 라벨 엣지 `A -. 보완 .-> B` 로 압축, 세부는
    `.fpanel` 패널로 보완 — 내용은 빈약해지지 않게 유지). 슬라이드 6 라이프사이클이 예시:
    분기 다이아몬드를 없애고 `Build → Optimize → Operate → 지속 운영` 일렬 + 피드백 점선으로 단순화.
  - 확대: 렌더된 다이어그램은 컨테이너 폭(최대 1100px)을 꽉 채우도록 SVG 를 키우고, **클릭(또는
    우상단 "🔍 확대" 버튼)하면 라이트박스로 전체 화면 확대**된다(자동, ESC/클릭 닫기). 작아서 안 보이는
    문제를 이 둘로 해결한다.
  - 견고성: 로더가 다이어그램마다 `mermaid.parse()` 로 먼저 검증해 **실패하면 깨진 폭탄 SVG 대신
    조용한 폴백 노트**로 대체하고(`suppressErrorRendering`), 정상인 것만 렌더한다. 그래도 위 규칙을
    지켜 애초에 에러가 나지 않게 작성한다.
- **방사형(orbit) 허브-스포크**: 중심 + 주변 노드 구조는 `.forbit`(중심 `.ocenter`, 주변 `.onode` 6개를
  `left/top` %로 원형 배치, 점선 링 회전 애니메이션, `prefers-reduced-motion` 가드)로 표현한다.

복잡한 3D·차트 등 다른 라이브러리도 필요하면 같은 방식(프래그먼트에 마크업 + 셸에서 CDN 로드)으로 추가할 수 있다.

## `.frag` 디자인 시스템 (생성기 CSS 제공)

- 컨테이너: `frag--dark`(어두운 히어로) / `frag--light`(밝은 배경) / `frag--plain`.
  `frag--center` 세로 가운데, `.tc` 텍스트 가운데.
- 타이포: `.ftitle` 제목, `.fsub` 부제, `.feyebrow`(대문자 라벨), `.fkicker`(우상단 "NN / 29"),
  `.flead` 리드 문단. 제목은 `.ftitle` 기본 크기(`clamp(30px,4.2vw,52px)`, 자동 줄바꿈·균형)에
  맡기고, **인라인 `font-size` 로 과하게 키우지 않는다**(히어로도 최대 ~54–58px 권장 — `7vw`/`84px`
  같은 값은 일부 폭에서 너무 커진다).
- 레이아웃: `.fgrid.c2/.c3/.c4` 그리드(`.imgL`/`.imgR` = 이미지 칸을 넓게 잡는 2단),
  `.fcol`(+`.h`/`.down`/`.d`) 컬럼, `.fpanel` 패널.
- 요소: `ul.fbul` 불릿, `.chips`/`.chip` 칩, `.fflow`(+`.node`/`.arr`) 가로 흐름,
  `.fstats`(+`.v`/`.l`) 수치, `.fstat-line`(+`<b>` 핑크 강조) 푸터, `.brandbar` 브랜드 로고.
- 이미지: `figure.fshot`(스크린샷 프레임, `.fshot--zoom` hover 확대) + `<img>` + `<figcaption>`.
- 다이어그램: `.fdiagram > pre.mermaid`(Mermaid), `.forbit`(방사형 허브-스포크).
- 색 토큰: `--msft-purple` `--msft-blue` `--msft-pink`. 텍스트색 기반 `color-mix` 라
  다크/라이트에 자동 적응.

## 인터랙션 · 애니메이션

HTML 의 강점을 살려 가볍게 더한다(과하지 않게).

- 스크롤 등장: 슬라이드는 스크롤 시 떠오르고(`.slide-wrap`), 프래그먼트 안에서 강조하고 싶은
  요소에 `class="reveal"` 을 주면 순차(stagger) 페이드업으로 등장한다.
- hover: `.fpanel`/`.fcol`/`.chip`/`.fflow .node`/`.fshot--zoom` 에 자동 hover 효과.
- 이미지 확대: `.fshot--zoom` 이미지와 **Mermaid 다이어그램**은 클릭 시 라이트박스로 전체 화면 확대(ESC·클릭으로 닫기) — 자동 적용.
- 접근성: `prefers-reduced-motion` 가드 내장 — 동작 최소화 설정이면 즉시 표시·효과 제거.

## 슬라이드 목차 (titles.json)

좌측 목차/번호에 쓸 제목을 `titles.json`(번호→제목) 으로 둔다. 없으면 추출 제목/`Slide N`.

```json
{ "1": "Microsoft Foundry 소개", "10": "모델 라우터" }
```

한 줄·25자 이내·명사형. 빠진 번호는 자동 처리.

## 영상

슬라이드에 영상이 있으면 HTML 에서 바로 재생된다.
- 프래그먼트 안에 YouTube/Vimeo 는 반응형 iframe, `.mp4`/`.webm`/`data:video` 는 `<video controls>`.
- PPTX 에 내장·링크된 영상은 프래그먼트가 없어도 **자동 추출**되어 재생된다.
- 애니메이션 GIF 는 프레임 보존(WebP 변환 안 함).

## 결과물 형태

- 좌측 사이드바: 슬라이드 목차 — scroll-spy 하이라이트, 클릭 시 부드러운 스크롤, 접기/펼치기 토글
- 본문: 위 → 아래로 세로 스크롤되는 슬라이드. 본문은 모델이 만든 HTML
- 글자 크기 `clamp()` 반응형, 라이트/다크 자동, 모바일 사이드바 토글
- 같은 자료를 다시 빌드하면 과거 버전 HTML 은 보존되고 새 버전이 추가된다(버전 전환 UI)

## 실행

```bash
python3 scripts/build_scroll_deck.py --releases <입력폴더> --out <출력폴더>
```

- 입력 폴더(릴리즈) 안에 PPTX 와 함께 `slides/NN.html`·`titles.json`·`slide-NN.png`·`slides/img/` 를 둔다.
- 출력 폴더는 고정하지 않는다 — 사용자가 원하는 경로(`--out`)로 만든다.
