# Scroll Deck Skill

**목적: PPT(PPTX) 를 입력으로 받아, 발표용 스크롤 HTML 로 만든다. 그게 전부다.**

원본 PPT 의 UI 를 그대로 복제하지 않는다. 슬라이드의 의도를 읽고 **고객 전달용 HTML 로
다시 구성**한다. 좌측에 슬라이드 목차, 위 → 아래로 스크롤되며 설명하는 한 장짜리 웹페이지.

> 파이프라인: **① PPT 분석 → ② 스크린샷으로 레이아웃·내용 파악 → ③ 모델(나)이 각 슬라이드를
> 깔끔한 HTML 로 재구성 → ④ 생성기가 목차·스크롤·영상 셸에 조립.**

생성기: `scripts/build_scroll_deck.py` (순수 Python + `python-pptx`, 새 의존성 없음)

## 작업 방식

1. **분석**: PPTX 본문과 슬라이드 스크린샷(`slide-NN.png`)으로 각 장표의 레이아웃·강조점·
   상세 콘텐츠와 정확한 텍스트를 파악한다.
2. **재구성**: 각 슬라이드를 `slides/NN.html`(2자리 zero-pad) 에 자체 완결형
   `<section class="frag ...">` HTML 로 작성한다. 원본을 억지로 모사하지 말고, 발표·고객
   전달에 맞게 깔끔히 재배치한다. 제목은 프래그먼트 안에서 직접 표현(자동 카드 헤더 없음).
   제품·기능명은 영문 유지.
3. **빌드**: 생성기를 실행하면 프래그먼트가 슬라이드 본문으로 삽입되고
   (`<div class="slide-wrap" id="slide-N">{프래그먼트}</div>`), 목차·스크롤·영상 셸이 조립된다.

> 스크린샷이 없으면 PPTX 본문만으로 작성한다. 프래그먼트가 없는 슬라이드는 생성기가
> PPTX 에서 본문을 자동 추출해 폴백 렌더한다(차선책 — 가능하면 프래그먼트로 재구성).

**원본보다 빈약하면 안 된다.** 텍스트만 복붙하지 말고, 원본의 불릿·수치·흐름을 모두
살리거나 확장한다. 제품 UI 스크린샷·다이어그램처럼 **글로는 전달이 약한 장표는 PPT 의
실제 이미지를 가져와 임베드**하고, HTML 인 만큼 **스크롤 등장 애니메이션·hover 등 가벼운
인터랙션**을 더해 자연스럽게 만든다.

## 새 PPT 에 적용하는 절차 (어떤 PPT 든 동일)

이 스킬은 특정 자료에 묶여 있지 않다. **완전히 다른 PPTX 를 받아도** 아래만 반복하면
그 PPT 내용에 맞는 발표용 스크롤 HTML 이 나온다. 핵심은 4번(슬라이드 재구성)이며, 그 PPT 의
주제·구조·톤에 맞춰 매번 새로 작성한다(아래 디자인·다이어그램·발표 모드 규칙은 공통 도구).

1. **릴리즈 폴더 생성**: `releases/<YYYYMMDD-HHMMSS>-<발표제목>/` 을 만들고 그 안에
   `<발표제목>.pptx` 를 넣는다. 폴더명 접두 타임스탬프가 **버전 키**다 — 같은 제목이면
   타임스탬프 최신순으로 버전 히스토리가 쌓이고, 가장 최신이 `latest.html` 이 된다.
   (이미 있는 자료를 다시 빌드하려면 새 타임스탬프 폴더로 두면 **새 버전**으로 추가된다.)
2. **스크린샷 생성(작성 보조용)**: 모델이 레이아웃·강조점을 보고 재구성하려면 슬라이드
   이미지가 필요하다. `python3 scripts/render_ppt_previews.py <릴리즈>` 로 `slide-NN.png`
   를 만든다(런타임에 `libreoffice`·`poppler-utils` 필요). *이 PNG 는 빌드 입력이 아니라
   작성 참고용*이다. 도구가 없으면 PPTX 본문 텍스트만으로 작성해도 된다.
3. **필요 이미지 추출**: 글로 전달이 약한 화면/다이어그램은
   `python3 scripts/extract_slide_media.py <pptx> <릴리즈>/slides/img` 로 뽑아 둔다(꾸밈용·
   초소형 그림은 자동 제외). 쓰는 것만 남긴다.
4. **슬라이드 재구성(핵심 · 모델 작업)**: 각 슬라이드의 의도를 읽어
   `<릴리즈>/slides/NN.html`(2자리 zero-pad) 프래그먼트로 **그 PPT 내용에 맞게** 작성한다.
   원본 레이아웃을 모사하지 말고 고객 전달용으로 재배치한다. 표현 도구는 아래 섹션 공통:
   `.frag` 디자인 시스템 · 이미지 임베드 · 다이어그램(**HTML 우선, Mermaid 는 꼭 필요할 때만**) ·
   인터랙션/애니메이션. 테마(밝기·포인트색)는 소스 PPT 에서 **자동 추출**되니 토큰/클래스를 쓴다
   (아래 "테마" 섹션).
5. **목차 작성**: `<릴리즈>/titles.json`(번호→제목). 없으면 추출 제목/`Slide N` 폴백.
6. **빌드·확인**: `python3 scripts/build_scroll_deck.py --releases releases --out docs/decks`
   (사이트 전체는 `python3 scripts/build_site.py`). 로컬에서 `latest.html` 을 열어 점검.
7. **배포**: 커밋·push 하면 `pages.yml` 이 `docs/` 를 통째로 재생성·배포한다(아래 "배포" 참고).

### 입력 폴더 구조 (빌드 계약)

```
releases/<타임스탬프>-<제목>/
├─ <제목>.pptx          ← (필수) 본문·발표자 노트 추출원
├─ titles.json          ← (권장) 좌측 목차 제목 {"1":"…","2":"…"}
└─ slides/
   ├─ 01.html … NN.html ← (핵심) 모델이 작성한 슬라이드 프래그먼트
   └─ img/sNN-k.webp    ← (선택) 프래그먼트가 참조하는 임베드 이미지
```

- 빌드가 실제로 읽는 것은 **PPTX · titles.json · slides/** 뿐이다. `slide-NN.png`·PDF 는
  작성 보조 자산이라 빌드/배포에 포함하지 않아도 된다(레포 경량화).
- 프래그먼트가 없는 슬라이드는 생성기가 PPTX 본문으로 **자동 폴백** 렌더한다(차선책 —
  가능하면 모두 프래그먼트로 재구성).
- 발표자 노트는 PPTX 의 *슬라이드 노트*에서 자동 추출되므로 따로 작성할 필요 없다.

## PPT 이미지 임베드

제품 화면·대시보드·다이어그램 등 **꼭 필요한 이미지만** 골라 가져온다(꾸밈용 배경·작은
아이콘은 제외 — 아이콘은 이모지/칩으로 대체).

```bash
python3 scripts/extract_slide_media.py <pptx> <릴리즈>/slides/img
```

- 슬라이드별 그림을 `img/sNN-k.webp`(최적화 WebP)로 추출. 배경(면적 과대)·초소형 그림은 건너뛴다.
- 프래그먼트에서 상대경로로 참조: `<figure class="fshot fshot--zoom reveal">
  <img src="img/sNN-1.webp" loading="lazy"><figcaption>…</figcaption></figure>`.
- 빌드 시 생성기가 `slides/img/` 를 출력 `<slug>/img/` 로 복사하므로 상대경로가 그대로 해석된다.
- 불필요하게 추출된 그림은 지운다(쓰는 것만 남긴다).
- **이미지가 작아 보이지 않게**: 이미지+텍스트 2단은 `.fgrid.c2` 대신 **`.fgrid.imgR`**(이미지 오른쪽)
  /**`.fgrid.imgL`**(이미지 왼쪽)을 써 이미지 칸을 더 넓게(약 60%) 잡는다. `.fshot--zoom` 이미지는
  **클릭하면 라이트박스로 전체 확대**(자동, ESC/클릭 닫기)되므로 작은 화면에서도 디테일을 볼 수 있다.

## 다이어그램 — HTML 우선, Mermaid 는 꼭 필요할 때만

순서: **① 네이티브 HTML/CSS 로 표현 가능하면 그걸로** → ② HTML 로 깔끔히 담기 어려운
복잡한 그래프만 Mermaid → ③ 허브-스포크는 `.forbit`. 원본을 그대로 베끼지 말고 핵심
구조(흐름·반복·관계)를 읽어 다시 그린다.

- **HTML 우선(기본값)**: 선형 흐름·단계, 비교, 계층/그룹, 통계, 행렬은 Mermaid 가 필요 없다.
  - 가로/세로 흐름: `.fflow`(+`.node`/`.arr`) 또는 `.fgrid.c3/.c4` 패널을 화살표(→/⤵)로 연결.
  - 계층·파이프라인: `.fgrid` + `.fpanel`(단계 라벨 `.feyebrow`)로 좌→우 단계 구성.
  - 관계·매핑: 2단 `.fgrid.c2` 패널 대응, 표는 스코프드 `<style>` 테이블.
  - 장점: 폰트·색이 슬라이드와 100% 일치, 반응형, 항상 또렷, 파서 에러 없음. **HTML 로 충분하면
    Mermaid 를 쓰지 않는다.**
- **Mermaid 는 "진짜 복잡할 때만"**: 다대다 연결, 피드백 루프가 얽힌 폐회로, 분기/병합이 많은
  파이프라인처럼 HTML 박스+화살표로는 지저분해지는 경우에 한해 사용한다.
  생성기가 CDN 로드·소스 PPT 색 기반 테마(짙은 네이비 카드)를 적용한다:
  `<div class="fdiagram"><pre class="mermaid">…</pre></div>`.
  - 주의: `style` 지시문 색상에 **`rgba(...)` 금지**(쉼표가 파서를 깨뜨림) → 8자리 hex(`#0078d422`) 사용.
  - 노드 텍스트색은 다이어그램 카드(네이비)에 맞춰 **밝은색**(`color:#eaf2fb`)으로.
  - `end` 는 예약어 → 노드 id 로 쓰지 않는다.
  - 노드가 많으면 가로로 과하게 펼쳐져 **화면에서 너무 작게** 보인다 → **고수준 루프/흐름만
    다이어그램**으로 그리고(결정 분기·반복은 점선 라벨 엣지 `A -. 보완 .-> B` 로 압축, 세부는
    `.fpanel` 패널로 보완 — 내용은 빈약해지지 않게 유지).
  - 확대: 렌더된 다이어그램은 컨테이너 폭(최대 1100px)을 꽉 채우도록 SVG 를 키우고, **클릭(또는
    우상단 "🔍 확대" 버튼)하면 라이트박스로 전체 화면 확대**된다(자동, ESC/클릭 닫기).
  - 견고성: 로더가 다이어그램마다 `mermaid.parse()` 로 먼저 검증해 **실패하면 깨진 폭탄 SVG 대신
    조용한 폴백 노트**로 대체하고(`suppressErrorRendering`), 정상인 것만 렌더한다.
- **방사형(orbit) 허브-스포크**: 중심 + 주변 노드 구조는 `.forbit`(중심 `.ocenter`, 주변 `.onode` 6개를
  `left/top` %로 원형 배치, 점선 링 회전 애니메이션, `prefers-reduced-motion` 가드)로 표현한다.

복잡한 3D·차트 등 다른 라이브러리도 필요하면 같은 방식(프래그먼트에 마크업 + 셸에서 CDN 로드)으로 추가할 수 있다.

## 테마 — 소스 PPT 색을 따라간다 (자동)

덱 테마는 **고정 색이 아니라 소스 PPTX 에서 자동 추출**한다(`extract_theme()` →
`render_theme_css()`). PPT 마다 테마가 달라지므로, 원본과 비슷한 톤의 결과가 나온다.

- 추출 항목: 슬라이드 배경 밝기 분포 → **light/dark 기본 모드**, 가장 빈번한 채도 높은 색 →
  **포인트색(accent)**, 어두운 배경색 → **타이틀/섹션 네이비**, 밝은 배경색 → **본문 페이퍼**.
- 결과는 `.frag` 토큰(`--msft-blue`/`--msft-purple`/`--msft-pink`)과 `.frag--dark`/`.frag--light`
  배경, 셸 `--accent`, Mermaid 테마에 **버전 HTML 마다 주입**된다. 추출 실패 시 보라(Foundry) 폴백.
- **작성 가이드(중요)**: 토큰을 직접 색으로 박지 말고 위 토큰/클래스를 쓰면 자동으로 그 PPT 색이 된다.
  - 밝은(light) PPT 면: **본문 슬라이드는 `frag--light`**(페이퍼 배경 + 짙은 글씨), **타이틀·섹션
    구분·마무리만 `frag--dark`**(네이비 히어로). 전부 `frag--dark` 로 만들면 원본이 밝은데도 어두운
    덱이 되어 톤이 어긋난다 — 원본 배경 분포를 따른다.
  - 어두운(dark) PPT 면 반대로 본문도 `frag--dark` 중심.
  - 강조색은 `var(--msft-blue)` 등 토큰으로. 긍정 수치는 초록(`#34d399`)처럼 의미색만 직접 쓴다.
  - Mermaid 노드 텍스트는 네이비 카드 기준 **밝은색**(`color:#eaf2fb`).

## `.frag` 디자인 시스템 (생성기 CSS 제공)

- 컨테이너: `frag--dark`(어두운 히어로) / `frag--light`(밝은 배경) / `frag--plain`.
  `frag--center` 세로 가운데, `.tc` 텍스트 가운데.
- 타이포: `.ftitle` 제목, `.fsub` 부제, `.feyebrow`(대문자 라벨), `.fkicker`(우상단 "NN / 29"),
  `.flead` 리드 문단. 제목은 `.ftitle` 기본 크기(`clamp(30px,4.2vw,52px)`, 자동 줄바꿈·균형)에
  맡기고, **인라인 `font-size` 로 과하게 키우지 않는다**(히어로도 최대 ~54–58px 권장 — `7vw`/`84px`
  같은 값은 일부 폭에서 너무 커진다).
- 레이아웃: `.fgrid.c2/.c3/.c4` 그리드(`.imgL`/`.imgR` = 이미지 칸을 넓게 잡는 2단),
  `.fcol`(+`.h`/`.down`/`.d`) 컬럼, `.fpanel` 패널.
- 요소: `ul.fbul` 불릿, `.chips`/`.chip` 칩, `.fflow`(+`.node`/`.arr`) 가로 흐름,
  `.fstats`(+`.v`/`.l`) 수치, `.fstat-line`(+`<b>` 핑크 강조) 푸터, `.brandbar` 브랜드 로고.
- 이미지: `figure.fshot`(스크린샷 프레임, `.fshot--zoom` hover 확대) + `<img>` + `<figcaption>`.
- 다이어그램: `.fdiagram > pre.mermaid`(Mermaid), `.forbit`(방사형 허브-스포크).
- 색 토큰: `--msft-purple` `--msft-blue` `--msft-pink` — **소스 PPT 색으로 자동 주입**(위 "테마"
  섹션). 텍스트색 기반 `color-mix` 라 다크/라이트에 자동 적응.

## 인터랙션 · 애니메이션

HTML 의 강점을 살려 가볍게 더한다(과하지 않게).

- 스크롤 등장: 슬라이드는 스크롤 시 떠오르고(`.slide-wrap`), 프래그먼트 안에서 강조하고 싶은
  요소에 `class="reveal"` 을 주면 순차(stagger) 페이드업으로 등장한다.
- hover: `.fpanel`/`.fcol`/`.chip`/`.fflow .node`/`.fshot--zoom` 에 자동 hover 효과.
- 이미지 확대: `.fshot--zoom` 이미지와 **Mermaid 다이어그램**은 클릭 시 라이트박스로 전체 화면 확대(ESC·클릭으로 닫기) — 자동 적용.
- 접근성: `prefers-reduced-motion` 가드 내장 — 동작 최소화 설정이면 즉시 표시·효과 제거.

## 발표 모드 (컨트롤 바 · 전체화면 · 발표자 모드)

모든 덱 하단에 떠 있는 **컨트롤 바**(`.deckbar`, 평소 흐릿하다 hover 시 또렷)에서 발표를 제어한다.
전부 자동 주입되므로 프래그먼트에서 따로 만들 필요 없다.

- **이전 / 다음**: `‹` `›` 버튼·`←`/`→`·`PageUp/Down` 으로 슬라이드 이동. 가운데 `현재/전체` 카운터.
  스크롤 위치는 뷰포트 중앙을 점유한 슬라이드 기준으로 추적하며, 버튼 이동 중에는 잠깐
  스크롤-추적을 잠가(`navAt`) 연속 클릭 시 카운터가 튀지 않는다.
- **전체화면**(`⛶`): Fullscreen API 토글.
- **발표**(`▶`): 한 장씩 전체화면으로 보는 슬라이드쇼 오버레이(`.present`). 슬라이드를 1280px 기준으로
  렌더 후 `transform:scale` 로 16:9 무대에 꽉 맞춰(`fitInto`) PPT처럼 일관된 크기로 보인다.
  `←`/`→`/`Space` 이동, `Esc` 종료.
- **발표자 모드**(`🎤`): 새 창(`#present=presenter`)을 열어 **2화면**으로 발표.
  - 발표자 창: 현재 슬라이드 + 다음 슬라이드 미리보기 + **발표자 노트** + 경과 타이머 + 이전/다음.
  - 원래 창: 관객용으로 현재 슬라이드만 전체화면.
  - 두 창은 `BroadcastChannel('deck:<덱이름>')` 로 현재 슬라이드를 실시간 동기화한다.
- **발표자 노트**는 PPTX 의 *슬라이드 노트*(`slide.notes_slide`)에서 자동 추출해 페이지에 `window.__NOTES`
  로 임베드한다. 노트가 있으면 발표자 모드가 그대로 활용한다(별도 작성 불필요).

## 레이아웃 · 가독성 디테일

- **이미지가 작아 보이지 않게**: 좌우 분할은 `.fgrid.imgL`/`.imgR`(이미지측 1.7 비중)을 쓰고, 단독
  스크린샷은 전폭으로 넣되 과대 확대는 막혀(상한 1040px·가운데 정렬) 있다. 작은 raster 를 좁은 칸에
  넣지 말 것.
- **글자가 너무 크지 않게**: 타이틀은 `clamp(27px,3.5vw,44px)` 선을 넘기지 않는다(과거 52px → 44px 로
  하향). 프래그먼트에서 인라인 `font-size` 를 줄 때도 본문 16–20px, 제목 ~44px 상한을 지킨다.
- 발표/발표자 모드는 슬라이드를 무대에 꽉 맞추므로(`fitInto`) 큰 화면에서 이미지·도형이 시원하게 보인다.

## 슬라이드 비율 — 모든 슬라이드 16:9 고정 (자동)

- 모든 `.slide-wrap` 은 **요즘 파워포인트 표준인 16:9 비율로 고정**된다(`aspect-ratio:16/9`).
  즉 스크롤 덱의 모든 슬라이드가 정확히 같은 가로·세로 크기를 가진다 — 생성기가 자동 처리하므로
  프래그먼트에서 따로 높이를 지정할 필요 없다.
- 각 프래그먼트의 내용은 빌드된 페이지에서 `.fitwrap` 으로 감싸지고, **한 슬라이드(16:9)에 넘치면
  자동으로 축소(`transform:scale`)** 되어 잘리지 않는다. 발표/발표자 모드의 무대 맞춤과 동일한 안전장치다.
- 그래도 **한 슬라이드에는 한 화면 분량만** 담는 게 원칙이다. 내용이 많아 과하게 축소(예: scale<0.7)되면
  글자가 옆 슬라이드보다 작아져 보이므로, 그럴 땐 **불릿을 줄이거나 슬라이드를 둘로 나눈다.**
  (예: 단계 그리드 + 같은 내용의 핵심 포인트 패널을 둘 다 넣지 말고 하나로 압축.)
- 작성 시 대략 **세로 ≈ 가로 × 9/16** 안에 들어오도록 섹션 수(타이틀 + 본문 1~2블록)를 잡는다.

## 슬라이드 목차 (titles.json)

좌측 목차/번호에 쓸 제목을 `titles.json`(번호→제목) 으로 둔다. 없으면 추출 제목/`Slide N`.

```json
{ "1": "Microsoft Foundry 소개", "10": "모델 라우터" }
```

한 줄·25자 이내·명사형. 빠진 번호는 자동 처리.

## 영상

슬라이드에 영상이 있으면 HTML 에서 바로 재생된다.
- 프래그먼트 안에 YouTube/Vimeo 는 반응형 iframe, `.mp4`/`.webm`/`data:video` 는 `<video controls>`.
- PPTX 에 내장·링크된 영상은 프래그먼트가 없어도 **자동 추출**되어 재생된다.
- 애니메이션 GIF 는 프레임 보존(WebP 변환 안 함).

## 결과물 형태

- 좌측 사이드바: 슬라이드 목차 — scroll-spy 하이라이트, 클릭 시 부드러운 스크롤, 접기/펼치기 토글
- 본문: 위 → 아래로 세로 스크롤되는 슬라이드. 본문은 모델이 만든 HTML
- 하단 컨트롤 바: 이전/다음·전체화면·발표(슬라이드쇼)·발표자 모드(2화면+노트)
- 글자 크기 `clamp()` 반응형, 라이트/다크 자동, 모바일 사이드바 토글
- 같은 자료를 다시 빌드하면 과거 버전 HTML 은 보존되고 새 버전이 추가된다(버전 전환 UI)

## 실행

```bash
python3 scripts/build_scroll_deck.py --releases <입력폴더> --out <출력폴더>
```

- 입력 폴더(릴리즈) 안에 PPTX 와 함께 `slides/NN.html`·`titles.json`·`slide-NN.png`·`slides/img/` 를 둔다.
- 출력 폴더는 고정하지 않는다 — 사용자가 원하는 경로(`--out`)로 만든다.

## 배포 (GitHub Pages)

이 저장소는 `main`/`docs` 를 GitHub Pages 로 서빙하고, `pages.yml` 워크플로가 push 마다
`scripts/build_site.py` 로 `docs/` 를 **통째로 재생성**한다. 따라서 덱이 사이트 빌드에 통합돼
있어야 배포 시 보존된다(따로 커밋한 `docs/decks/` 는 워크플로가 덮어써 사라진다).

- **통합**: `build_site.py` 의 `build()` 끝에서 `build_scroll_deck.build(releases_dir, out_dir/"decks")`
  를 호출해 `docs/decks/` 를 함께 생성한다(인덱스에 "발표용 덱" 링크). 새 릴리즈를 올리면 워크플로가
  덱까지 자동 재빌드·배포한다.
- **Git LFS 주의**: PPTX·PDF 가 LFS 로 저장돼 있으면, CI 의 `actions/checkout` 에 **`lfs: true`** 가
  없으면 포인터(133B)만 받아 `python-pptx` 가 `Package not found` 로 실패한다. 워크플로 checkout 에
  `lfs: true` 를, 빌드 전에 `pip install -r scripts/requirements.txt`(python-pptx·Pillow) 를 둔다.
- 라이브 URL 예: `https://<user>.github.io/<repo>/decks/<슬러그>/latest.html`(슬러그는 한글 퍼센트 인코딩).
