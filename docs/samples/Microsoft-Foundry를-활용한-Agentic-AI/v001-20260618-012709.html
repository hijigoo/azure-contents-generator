에이전트 개발 라이프사이클

에이전트는 한 번 만들고 끝나지 않습니다. Build → Optimize → Operate 의 반복 사이클로 발전합니다.

1. Build

  • 모델·도구·에이전트 리서치
  • 도구와 지식 추가
  • 에이전트 빌드와 배포
  • 애플리케이션 통합

2. Optimize

  • 프롬프트 테스트와 평가
  • 모델 미세 조정
  • 코드 커스터마이징
  • 멀티 에이전트 오케스트레이션

3. Operate

  • 확장형 관리형 런타임 호스팅
  • 비용·규제 준수·안정성 모니터링
  • 구성 추적과 디버깅
  • 위험 완화 및 보호

스마트 라우팅의 가치

스마트 라우팅은 사용자 경험을 해치지 않으면서 품질을 유지하고 비용을 최적화합니다.

Challenge

작업에 맞는 LLM 선택은 어렵고, 사용자는 모든 사례에 단일 모델을 고집하기 쉽습니다.

Opportunity

최적 모델을 동적으로 선택하면 품질을 지키면서 비용을 줄일 수 있습니다.

  1. 1

    엔드포인트 호출

    사용자가 API로 라우터 엔드포인트를 호출합니다.

  2. 2

    동적 지능형 라우팅

    라우터가 프롬프트·도구 사용 등 입력을 평가해 최적 모델로 라우팅합니다.

  3. 3

    최적화된 응답 반환

    선택된 모델의 응답을 사용자에게 돌려줍니다.

예시 "5월 파리 날씨는?" 같은 간단한 질문은 경량 모델(예: GPT-4.1-nano)로, "비건 가족을 위한 5일 파리–베네치아 일정" 같은 복잡한 요청은 더 강력한 모델로 라우팅합니다.

Agent Service 배포 흐름

원하는 프레임워크로 만든 에이전트를 컨테이너로 패키징해 Foundry Agent Service에 배포합니다.

Agent frameworks → Build / Package → Containers → Deploy → Foundry Agent Service

배포된 Hosted Agent는 클라이언트의 호출을 받아 관리형 런타임 위에서 실행됩니다.

원클릭 퍼블리싱 (M365)

Evaluators 평가 지표

Foundry는 품질·안전·에이전트 동작·검색까지 폭넓은 평가 지표(Evaluators)를 기본 제공합니다.

Quality

  • Groundedness · Relevance
  • Coherence · Fluency · Similarity
  • NLP Metrics (예: F1)

Risk & Safety

  • Hate · Sexual · Violence · Self-Harm
  • Jailbreak / Indirect Attack
  • Protected Material · Code Vulnerability

Agents

  • Intent Resolution
  • Tool Call / Selection Accuracy
  • Task Adherence · Completion
기본 제공 평가자 외에 Custom Evaluators 로 조직 고유의 기준도 추가할 수 있습니다.

에이전트 평가 예시

에이전트 평가는 실제 실행 흐름의 각 단계를 지표로 점검합니다 — 예: 일기예보 요청.

  1. 1

    User Intent 분석

    "샌프란시스코 날씨는 어때?" → 지역·시간 의도 파악 (의도 해석 평가)

  2. 2

    도구 호출

    get_loc() → get_weather(loc, time) (도구 선택·입력·출력 정확도)

  3. 3

    응답

    "흐리고 화씨 60도, 비 가능성" (과제 준수·완료율 등 운영 지표)

모델 커스터마이징 옵션

모델 적응의 성숙도에 따라 단계적으로 가치를 높여갑니다 — 베이스 모델에서 지속 개선 루프까지.

  1. 1

    Base model

    Foundry Models 의 기본 모델로 시작

  2. 2

    Prompt engineering

    선언적(Declarative) 에이전트 구성

  3. 3

    RAG · Tool calling

    지식 검색과 멀티 에이전트, Foundry IQ 활용

  4. 4

    Fine-tuning · RAFT

    Foundry 내 미세 조정·증류(distillation)

  5. 5

    Continuous improvement

    피드백·신호 루프로 지속 개선, Control Plane 으로 운영

오른쪽 단계로 갈수록 모델 적응 성숙도와 비즈니스 가치가 함께 커집니다.
