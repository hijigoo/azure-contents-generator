모델 라우터 동작 구조

Challenge

작업에 적합한 LLM을 선택하는 것은 어렵습니다. 사용자들은 특정 사용 사례에 대해 단일 LLM에 고집하는 경우가 많습니다.

Opportunity

스마트 라우팅은 품질을 유지하면서 비용을 최적화합니다. 최적의 모델을 동적으로 선택해 사용자 경험을 저해하지 않으면서 비용을 절감합니다.

1 단계

사용자가 API로 엔드포인트 호출

  • 간단한 질문 — "5월 파리 날씨는 어때?"
  • 복잡한 쿼리 — "비건 가족을 위한 5일 일정 계획"
2 단계

라우터가 최적 모델로 동적 라우팅

  • 입력 매개변수(프롬프트·도구 사용)를 평가
  • GPT-4.1-nano · GPT-4.1-mini · … 중 선택
3 단계

최적화된 응답 반환

  • 선택된 모델의 응답을 그대로 반환
  • 최적화된 Response 제공

13 / 29

멀티 에이전트 워크플로우

Foundry Agent Service 의 빌더에서 에이전트를 구성하고, 바로 옆 Playground에서 테스트합니다.

에이전트 구성 단계
  • 모델 선택 — 작업에 맞는 모델 지정 (예: gpt-4.1-mini)
  • 프롬프트 작성 — 에이전트 지침(Instructions) 정의
  • 도구 연결 — Tools 로 외부 기능 연동
  • 지식 연결 — Knowledge 로 데이터 그라운딩
  • 메모리 연결 — 대화·상태 컨텍스트 유지
  • 가드레일 연결 — Guardrails 로 안전성 확보

구성한 에이전트는 우측 Playground에서 즉시 채팅으로 검증하고, Traces · Monitor · Evaluation 으로 동작을 추적합니다.
