ExpressRoute Peering 비교

Private Peering · Microsoft Peering 구분과 활용 시나리오

🔒 Private Peering
  • 연결 대상 — Azure VNet 내부 IaaS·PaaS (Private IP): VM, AKS, Storage PE, SQL MI
  • RFC1918 사설 IP 그대로, NAT 불필요 · On-prem ↔ VNet 완전 사설 통신
  • DC 확장(L3) 개념, Site-to-Site VPN 대체 · Latency 5~30 ms
  • 최적 — 센서 적재, AML Training 접근, VDI, ERP, 핵심 업무 트래픽
🌐 Microsoft Peering
  • 연결 대상 — Microsoft Public Services (Public IP): Azure PaaS Public EP
  • Public IP/DNS 해석 필요, NAT 가능 · Route Filter로 서비스/리전 선택
  • 인터넷 우회로 M365 품질·보안 향상 · 글로벌 prefix는 ER Premium
  • 최적 — Blob Public EP, Public API

권장 — ER 회선 1개에 Private + (선택) Microsoft Peering 동시 활성화 → IaaS/PaaS + M365/SaaS를 단일 회선으로 처리, Private Endpoint Inbound Hidden Cost 절감
