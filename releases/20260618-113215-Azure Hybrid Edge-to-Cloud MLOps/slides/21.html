Azure IoT Edge 디바이스에서의 ML 추론

Azure ML 학습·저장 → IoT Hub Module Twin 배포 → 모델 파일 Edge 다운로드 → Edge 모듈 로컬 추론(Web API)

  1. Azure ML → Blob — 학습 후 모델 파일 저장
  2. IoT Hub → Module Twin — 배포 구성(모델 메타) 전달
  3. HTTP 다운로드 — Blob → 공유 로컬 스토리지
  4. 로컬 복사 — AI Loader 모듈 저장소로
  5. 모델 로드 — *.tflite · *.onnx 메모리 로드
  6. Client → Web API — 로컬 엔드포인트 현장 추론

AI Loader 구성 — Python·Flask · OpenCV/PIL · IoT device SDK · TFLite/ONNX Runtime · IoT Edge Runtime(agent·hub)

OTA 교체 = Blob 업로드 + Module Twin 변경 두 단계 → 무중단 원격 업데이트
