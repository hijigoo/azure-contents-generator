Azure IoT Edge 디바이스에서의 ML 추론

Azure ML 학습·저장 → IoT Hub Module Twin 배포 구성 → 모델 파일 Edge 다운로드 → Edge 모듈 로컬 추론(Web API)

주요 데이터 흐름
  1. Azure ML → Blob — 모델 학습 후 모델 파일 저장
  2. IoT Hub → Module Twin — Edge 모듈에 배포 구성(모델 메타) 전달
  3. HTTP 다운로드 — Blob → 공유 로컬 스토리지로 모델 내려받기
  4. 공유 → 모듈 로컬 — AI Loader 모듈 저장소로 복사
  5. 모델 파일 로드 — *.tflite · *.onnx 를 Edge 모듈에 로드
  6. Client → Web API — 현장 추론 요청·응답 (로컬 엔드포인트)
Edge 모듈(AI Loader) 구성
Python AI Loader · Flask Web API Module Twin · OpenCV · PIL Azure IoT device SDK TensorFlow Lite / ONNX Runtime Linux 컨테이너 · IoT Edge Runtime (agent·hub)
